# CS-320-Software-Test

Functional and secure software starts with building the program around clear requirements and then testing it carefully. In this course, I used unit tests to confirm that each part of the program worked the way it was supposed to. Writing tests for the contact service helped me verify that valid data was accepted and invalid data was rejected. Security is also part of making software functional because a program is not truly successful if it allows bad input, crashes easily, or exposes data in unsafe ways. I can help ensure software is functional and secure by validating inputs, following requirements closely, testing normal and invalid cases, and reviewing code for possible weaknesses before release.

To interpret user needs and incorporate them into a program, I start by looking closely at the requirements and identifying what the user or client actually expects the software to do. I break those needs into smaller parts, such as features, rules, and limitations, and then translate them into code and test cases. For example, with the contact service, the requirements defined things like field length, required values, and update rules. I used those requirements to guide both the design of the classes and the tests I created. This helped make sure the final product matched the intended use instead of just what I thought it should do.

When I approach designing software, I begin with the problem the program is meant to solve. Then I organize the solution into smaller, manageable pieces so the code is easier to build, test, and maintain. In this course, that meant separating the contact object from the contact service and then creating tests for both. I try to design software so that each class has a clear purpose and the program can be expanded or updated later without rewriting everything. My approach to design focuses on simplicity, meeting requirements, and supporting quality through strong testing. Over time, I have learned that good software design is not just about making code work, but about making it understandable, reliable, and easier to improve.
